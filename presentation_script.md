# 🎙️ สคริปต์นำเสนอระบบฐานข้อมูลเครือข่ายศิษย์เก่า (Alumni Network System)

*คำแนะนำ: ข้อความในวงเล็บ `[...]` คือสิ่งที่ผู้พูดต้องกระทำหรือกดบนหน้าจอ ส่วนข้อความปกติคือบทพูดครับ*

---

## 🏁 กล่าวเปิดงาน (Introduction)
**[เปิดหน้า Home Page ค้างไว้]**

"สวัสดีท่านคณะกรรมการทุกท่านครับ วันนี้ผมจะขอนำเสนอระบบฐานข้อมูลและเครือข่ายศิษย์เก่า ซึ่งจุดเด่นของระบบเราคือการใช้เทคโนโลยีฐานข้อมูลแบบคู่ขนาน (Hybrid Database) นั่นคือใช้ **PostgreSQL** สำหรับเก็บข้อมูลประวัติพื้นฐานเพื่อการค้นหาที่แม่นยำ และใช้ **Neo4j (Graph Database)** สำหรับสร้างเครือข่ายความสัมพันธ์ครับ ขออนุญาตเริ่มสาธิตการทำงานของระบบครับ"

---

## 👤 ช่วงที่ 1: การเข้าสู่ระบบและจัดการบัญชี (Authentication)

**[คลิกไปที่หน้า สมัครสมาชิก - Register]**
"เริ่มจากฝั่งศิษย์เก่า หากยังไม่มีบัญชี สามารถเข้ามากรอกรหัสนักศึกษา อีเมล รหัสผ่าน พร้อมกับประวัติการศึกษาและการทำงานได้ที่หน้านี้ครับ ทันทีที่ผู้ใช้กดปุ่ม 'สมัครสมาชิก' ข้อมูลจะถูกบันทึกลงฐานข้อมูลปกติ และจะถูกนำไปวาดเป็นโหนด (Node) เพื่อสร้างเส้นความสัมพันธ์ใน Graph Database โดยอัตโนมัติทันทีครับ"

**[คลิกไปที่หน้า เข้าสู่ระบบ - Login]**
"สำหรับผู้ที่มีบัญชีอยู่แล้ว สามารถเข้าสู่ระบบด้วยรหัสนักศึกษาและรหัสผ่านได้เลยครับ และหากจำรหัสผ่านไม่ได้ เราก็มีฟังก์ชัน **ลืมรหัสผ่าน (Forgot Password)** เพื่อให้ผู้ใช้งานสามารถกู้คืนบัญชีได้อย่างปลอดภัยครับ"

*(ทำการ Log in เข้าสู่ระบบด้วยบัญชีศิษย์เก่า)*

---

## 🎓 ช่วงที่ 2: ฝั่งผู้ใช้งานทั่วไป (Alumni User)

**[หน้าหลัก - Home]**
"เมื่อล็อกอินเข้ามา จะพบกับหน้าจอหลัก (Dashboard) ที่แสดงภาพรวมของระบบ เมนูต่างๆ และโพสต์ข่าวสารล่าสุด เพื่อให้ศิษย์เก่าอัปเดตข้อมูลได้อย่างรวดเร็วครับ"

**[คลิกไปที่เมนู แหล่งแชร์ความรู้ - Knowledge]**
"ถัดมาคือเมนูแหล่งแชร์ความรู้ ซึ่งจะเป็นพื้นที่สำหรับอ่านบทความ ข่าวสาร หรือประกาศรับสมัครงานต่างๆ ที่แอดมินนำมาแชร์ไว้ ผู้ใช้สามารถเลื่อนดูและคลิกอ่านรายละเอียดได้ครับ"

**[คลิกไปที่เมนู ค้นหาศิษย์เก่า - Search]**
"หากต้องการหาเพื่อนหรือรุ่นพี่แบบเจาะจง สามารถใช้เมนูค้นหานี้ได้ครับ หน้านี้ดึงข้อมูลจาก PostgreSQL โดยตรง ทำให้สามารถกรองข้อมูลได้แม่นยำและรวดเร็ว เช่น เราสามารถเลือกกรองเฉพาะ 'คณะวิทยาศาสตร์และเทคโนโลยี' หรือกรองตาม 'ตำแหน่งงาน' ระบบก็จะลิสต์รายชื่อออกมาเป็นรูปแบบการ์ดให้ติดต่อได้ทันทีครับ"

**[คลิกไปที่เมนู เครือข่าย - Graph Tracking]**
"และนี่คือฟังก์ชันไฮไลท์ของระบบเราครับ เมนูเครือข่าย (Graph Tracking) 
**[ใช้เมาส์ซูมเข้า/ออก เลื่อนกราฟให้กรรมการดู]**
เมื่อกดเข้ามา เราจะเห็นภาพรวมของเครือข่ายศิษย์เก่าทั้งหมดที่ดึงมาจาก Neo4j ครับ 

**[คลิกที่ Node รูปคน 1 อัน]**
เมื่อเราคลิกที่รายชื่อใครสักคน แผงด้านขวาจะบอกรายละเอียดโปรไฟล์ของเขา และด้านล่างจะแสดง **'รายการเชื่อมโยง'** ให้เห็นชัดเจนเลยว่า คนๆ นี้ รู้จักใครบ้าง, เรียนสาขาอะไร, และทำงานที่บริษัทไหน 

**[คลิกที่รายชื่อบริษัท/สาขา ในแผงด้านขวา]**
ซึ่งถ้าเราสนใจบริษัทหรือสาขานี้ เราสามารถคลิกจากในรายการได้เลย ระบบก็จะกระโดดพาเราไปโฟกัสที่จุดนั้น เพื่อดูว่ามีใครที่ทำงานหรือเรียนที่นี่อยู่อีกบ้าง ทำให้การสร้างคอนเนคชันทำได้ง่ายและเห็นภาพรวมอย่างชัดเจนครับ"

---

## ⚙️ ช่วงที่ 3: ฝั่งผู้ดูแลระบบ (Admin)

"ต่อไปผมขออนุญาตสลับบัญชี เข้าสู่ระบบในฐานะ **ผู้ดูแลระบบ (Admin)** เพื่อแสดงระบบจัดการหลังบ้านนะครับ"
*(ทำการ Log in ด้วยบัญชี Admin)*

**[คลิกไปที่เมนู สร้างโพสต์ - Create Post]**
"เริ่มจากระบบประกาศข่าวสาร แอดมินจะมีสิทธิ์เข้าถึงหน้า **สร้างโพสต์** สามารถเพิ่มหัวข้อข่าว เลือกหมวดหมู่ อัปโหลดรูปภาพหน้าปก และพิมพ์เนื้อหาเพื่อเผยแพร่ไปยังหน้าแหล่งแชร์ความรู้ให้ศิษย์เก่าเข้ามาอ่านได้ครับ"

**[คลิกไปที่เมนู แดชบอร์ดแอดมิน - Admin Dashboard]**
"และส่วนที่สำคัญที่สุดคือ Admin Dashboard ครับ หน้านี้เป็นศูนย์ควบคุมทั้งหมด แบ่งการทำงานหลักๆ ดังนี้ครับ:

1. **สถิติภาพรวม (Overview Stats):** **[ชี้ที่กราฟ/ตัวเลขด้านบน]** แสดงตัวเลขผู้ใช้งานทั้งหมด, สถิติการสมัครใหม่ และกราฟสถิติต่างๆ แบบ Real-time
2. **จัดการผู้ใช้งาน (User Management):** **[เลื่อนลงมาที่ตารางรายชื่อ]** แอดมินสามารถดูรายชื่อผู้ใช้ทั้งหมด ค้นหาตัวบุคคล และมีสิทธิ์คลิกเพื่อ **แก้ไขข้อมูล** หรือ **ลบบัญชี** ผู้ใช้ได้จากตารางนี้เลยครับ
3. **สถานะฐานข้อมูลกราฟ (Graph Database Audit):** **[คลิกแท็บ Neo4j หรือชี้ที่สถานะ DB]** เนื่องจากเราใช้ DB สองตัวคู่กัน แอดมินจึงสามารถตรวจสอบสถานะการเชื่อมต่อของ Neo4j จำนวนโหนด และจำนวนเส้นความสัมพันธ์ทั้งหมดได้
4. **ระบบซิงค์ข้อมูล (Sync Tools):** **[ชี้ที่ปุ่ม Sync / Cleanup]** หากเกิดเหตุขัดข้องหรือข้อมูลใน 2 ฐานข้อมูลไม่ตรงกัน แอดมินสามารถกดปุ่ม **'Sync All Users'** เพื่อสั่งให้ระบบวาดกราฟใหม่ให้ตรงกับประวัติในฐานข้อมูลตาราง หรือกด **'ลบ Node ขยะ'** เพื่อเคลียร์โหนดที่ไม่มีคนเชื่อมต่อแล้วทิ้ง ทำให้กราฟสะอาดและมีความถูกต้องตลอดเวลาครับ"

---

## 🎯 กล่าวปิดการพรีเซนต์ (Conclusion)

"ด้วยสถาปัตยกรรมระบบทั้งหมดนี้ ทำให้แพลตฟอร์มของเราไม่เพียงแต่เก็บข้อมูลศิษย์เก่าได้อย่างเป็นระเบียบ แต่ยังสามารถนำข้อมูลเหล่านั้นมาสร้างเป็นมูลค่า แสดงให้เห็นเป็นเครือข่ายความสัมพันธ์ที่ใช้งานได้จริง เพื่อให้ศิษย์เก่าได้กลับมาสร้างคอนเนคชันและช่วยเหลือกันต่อไปครับ 

ขอจบการนำเสนอระบบไว้เพียงเท่านี้ครับ ขอเชิญคณะกรรมการซักถามได้เลยครับ ขอบคุณครับ"
